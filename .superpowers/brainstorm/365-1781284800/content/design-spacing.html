Design — Part 2: Spacing System

A single layout rule applied to every section — so the site has a consistent visual rhythm as you scroll.

The Core Rule

Every section follows this two-layer pattern:

<section className="py-16 md:py-24 bg-[color]">
  <div className="container">
    {/* content here */}
  </div>
</section>
  • py-16 md:py-24 — consistent top/bottom breathing room (64px mobile → 96px desktop)
  • container — already defined in index.css as max-w-7xl mx-auto px-4 sm:px-6 lg:px-8
  • Hero stays full-bleed (special case) — but its inner content box moves into the container

What changes in tailwind.config.js

// tailwind.config.js — add these spacing tokens
theme: {
  extend: {
    spacing: {
      'section-sm': '4rem',   // py-section-sm → 64px (mobile)
      'section-md': '6rem',   // py-section-md → 96px (desktop)
    }
    // existing colors + fonts stay unchanged
  }
}

Optional — we can also just use py-16 md:py-24 directly without custom tokens. Tokens are cleaner if the team wants to change the rhythm site-wide later in one place.
