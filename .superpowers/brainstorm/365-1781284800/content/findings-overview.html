ZIT Site — Diagnostic Findings

Here's what I found after analyzing the full codebase. Let me know which area matters most to tackle first.

🔴 Critical Performance Issues

Three.js loaded globally
@react-three/fiber + three ≈ 600KB–1.4MB added to your bundle — even on pages that never use 3D
Daily.js loaded in dependencies
@daily-co/daily-js is a ~2MB video conferencing library sitting in your deps — likely unused on the site
Stripe initialized twice
loadStripe() is called in both main.tsx and App.tsx — loads the Stripe script twice on every page
PayPal + Stripe wrap entire app
Both payment providers initialize on every page visit, not just the donation page — unnecessary network overhead

🟠 Image & Asset Issues

No lazy loading on images
80+ image assets, none have loading="lazy" — all downloaded immediately on page load
No WebP format
All images are raw .jpg / .jpeg — no WebP conversion means 30–50% larger file sizes
Hero uses CSS background-image swap
Slideshow uses style.backgroundImage transition — forces full repaint on every slide change. Not GPU-accelerated
No Suspense fallback in App.tsx
Lazy-loaded routes have no <Suspense> boundary — can cause blank screens while pages load

🟡 UI Spacing Inconsistencies

Section Horizontal Padding Vertical Padding Layout System
HeroSection md:px-31 ❌ invalid class h-[90vh] ad-hoc %
TTM Section px-4 md:px-28 py-20 raw padding
TutorshipSection px-6 md:px-16 lg:px-28 py-16 raw padding
HowWeTeach container mx-auto px-4 py-6 pb-14 (via .section) container
StudySucceed px-4 + container py-16 mixed

Result: sections look visually misaligned — some text runs edge-to-edge, others sit in a narrow centered column, creating a jagged rhythm as you scroll.
